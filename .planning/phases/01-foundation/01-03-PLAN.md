Create `.github/workflows/deploy.yml`. The workflow must be in the repo root `.github/workflows/` directory — NOT inside `noesis/`.

```yaml
name: Deploy Noesis to GitHub Pages

on:
  push:
    branches:
      - main
  # Allow manual trigger from Actions tab
  workflow_dispatch:

# Grant permissions for GitHub Pages deployment
permissions:
  contents: read
  pages: write
  id-token: write

# Only allow one concurrent deployment; cancel in-progress runs
concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: noesis
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Node.js 20
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'
          cache-dependency-path: noesis/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Build Next.js static export
        run: npm run build
        env:
          # Set basePath to repo name for GitHub Pages subdirectory deployment
          # For custom domain or Vercel: set to '' or leave unset — zero code change required (INFRA-03)
          NEXT_PUBLIC_BASE_PATH: /NotebookLM

      - name: Upload Pages artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./noesis/out

  deploy:
    runs-on: ubuntu-latest
    needs: build
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

**Important notes on this workflow:**
- `working-directory: noesis` applies to all `run` steps in the build job, so `npm ci` and `npm run build` run inside `noesis/`
- `path: ./noesis/out` in the upload step is relative to the repo root (not working-directory), so it must be prefixed with `noesis/`
- `NEXT_PUBLIC_BASE_PATH: /NotebookLM` uses the GitHub repo name. If the repo is renamed, update this value — no code changes needed (INFRA-03 satisfied)
- The `concurrency` block prevents two simultaneous deployments racing to GitHub Pages

Create `noesis/.nojekyll` — an empty file that tells GitHub Pages to skip Jekyll processing (which would break Next.js routing):

```
(empty file — just create it with no content)
```

The `.nojekyll` file must live in the `out/` directory at build time OR be included in the artifact. The cleanest approach: add it to the Next.js `public/` directory so Next.js copies it to `out/` during export.

Create `noesis/public/.nojekyll` (empty file). This ensures it is always included in the static output.

Also create `noesis/.nojekyll` at the noesis root as a fallback reference.
