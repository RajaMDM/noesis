  <how-to-verify>
    Wait for Claude to report automated test results and confirm the git push. Then proceed with the visual checks below.

    **Step 1 — Get the public URL**
    Visit: `https://github.com/{owner}/NotebookLM/actions`
    Find the latest "Deploy Noesis to GitHub Pages" run -> click it -> find the deployed URL in the deploy step output.
    It should look like: `https://{owner}.github.io/NotebookLM/`

    **If GitHub Pages is not yet enabled:**
    Go to: Repository Settings -> Pages -> Source -> select "GitHub Actions"
    Then re-run the workflow from the Actions tab.

    **Step 2 — Aesthetic check (DSGN-01)**
    Open the URL in your browser. Evaluate:
    - [ ] Background is true dark — near-black (#080808 range), NOT soft gray or blue-tinted
    - [ ] Accent color is electric blue/cyan — visible on CTA button and topic card icons
    - [ ] Topic cards have a visible frosted-glass appearance — subtle transparency, 1px border, backdrop blur
    - [ ] Typography feels premium — crisp Inter for body, no fallback system fonts visible
    - [ ] Overall feel: Vercel / Linear aesthetic — NOT colorful ed-tech (no bright backgrounds, no Duolingo energy)
    - [ ] "What makes Noesis different" section renders with glassmorphism cards, icons, and concise copy

    **Step 3 — Animation check (DSGN-01)**
    Reload the page and verify:
    - [ ] Hero headline and subtitle fade/slide in on page load
    - [ ] Hover over a topic card — it lifts upward (~8px) with smooth transition
    - [ ] Scroll down slowly — hero background has subtle parallax (shifts slightly as you scroll)
    - [ ] Topic cards animate in as you scroll into view (staggered, not all at once)

    **Step 4 — Responsive check (DSGN-02)**
    Resize browser to 390px width (or use DevTools -> iPhone 14):
    - [ ] Hamburger menu icon is visible in top right of nav
    - [ ] Click hamburger — mobile menu opens with all 7 topic names listed
    - [ ] Topic cards stack to a single column
    - [ ] Hero text fits without horizontal overflow
    - [ ] CTA buttons are fully visible, not clipped

    Resize to 768px (tablet):
    - [ ] Topic cards show 2 per row
    - [ ] Nav shows the Topics button (not hamburger) — OR hamburger still shows at 768px (either is acceptable)

    **Step 5 — CI/CD verification (INFRA-02)**
    Make a trivial change (e.g., update footer text), commit, and push to main:
    ```bash
    # In noesis/src/app/page.tsx, change footer text slightly
    git add . && git commit -m "test: verify CI/CD triggers on push" && git push origin main
    ```
    - [ ] GitHub Actions workflow starts within ~30 seconds of push
    - [ ] Build completes successfully (green checkmark)
    - [ ] Deploy completes and updated site is live at the same URL

    **Step 6 — Navigation check**
    - [ ] Click "Data Sources" topic card -> loads `/topics/data-sources/` stub page
    - [ ] Topic page shows the topic title, icon, description, and "Content coming in Phase 2" placeholder
    - [ ] "Back to all topics" link returns to homepage
    - [ ] Visit a non-existent URL (e.g., `/topics/fake-topic`) -> shows custom 404 page
  </how-to-verify>
