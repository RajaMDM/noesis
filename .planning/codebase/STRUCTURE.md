## Naming Conventions

**Files:**
- Python scripts: `snake_case.py` (e.g., `yt_research.py`, `notebooklm_pipeline.py`)
- Generated artifacts: `{artifact_type}_{timestamp}.{extension}` (e.g., `infographic_20260315_075900.png`, `slides_20260315_072139.pdf`)

**Directories:**
- Application directories: `snake_case` (e.g., `yt-research`, `notebooklm`, `output`)
- Hidden/config directories: `.{name}` (e.g., `.claude`, `.git`, `.planning`)

**Functions:**
- Async operations: Prefixed with `async def` (e.g., `async def create_notebook()`, `async def add_youtube_sources()`)
- Helper functions: `snake_case` (e.g., `search_youtube()`, `format_results_table()`, `log()`, `timestamp()`)

**Variables:**
- Module-level constants: `UPPER_SNAKE_CASE` (e.g., `OUTPUT_DIR`)
- Enum mapping constants: `_UPPER_SNAKE_CASE` (e.g., `_STYLE_MAP`, `_ORIENTATION_MAP`) - leading underscore indicates internal use
- Local variables: `snake_case` (e.g., `ydl_opts`, `results`, `notebook_id`)

**Types:**
- Type hints used consistently in function signatures (Python 3.10+ union syntax with `|`)
- Examples: `list[dict]`, `str`, `dict`, `list[str] | None`

- Success: Prints formatted output to stdout; metadata to stderr; exits with code 0
- Error: Prints error message to stderr; exits with code 1 (line 133)

- Success: Prints JSON results to stdout; exits with code 0
- Error: Prints error to stderr; exits with code 1 (line 361)
